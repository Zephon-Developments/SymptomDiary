# SymptomDiary Plan Documents

## Purpose

Plan files track active work and current progress in a repo-local, resumable form.

## Location

- Store active plans in `Documentation/Plans/`.

## Naming Convention

- Use one active plan file per initiative.
- Preferred filename: `Plan_<topic>.md`.
- Update the existing file as work progresses rather than creating unnecessary new versions.

## Required Plan Sections

- Objective
- Constraints
- Assumptions
- Stage breakdown
- Acceptance criteria
- Validation strategy
- Risks and mitigations

## Required Progress Sheet

| Stage | Owner | Status | Completion Criteria | Notes |
| --- | --- | --- | --- | --- |

## Allowed Status Values

- Not Started
- In Progress
- Blocked
- Complete

## Maintenance Rule

- Update the plan whenever a stage starts, blocks, completes, or changes scope.
